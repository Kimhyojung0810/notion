# Notion 마감일 댓글 알림 — 운영·공유 가이드

이 문서는 **팀·회사 내 공유**를 위해 정리한 것입니다. Notion DB 스키마, 알림 정책, 보안·배포 절차를 한곳에서 확인할 수 있습니다.

---

## 1. 이 도구가 하는 일

1. 지정한 Notion **데이터베이스**의 모든 행(페이지)을 조회합니다.  
2. **마감일·상태** 등 조건에 맞으면, 해당 **페이지의 댓글**로 알림 문구를 남깁니다.  
3. `Person`에 사람이 있으면 댓글 앞에 **@멘션**을 넣습니다.  
4. 실행은 **GitHub Actions**에서 **매일 한국 시간 10시**에 자동으로 돌리는 것을 기본으로 합니다.

**하지 않는 것**

- 슬랙/이메일로 직접 발송 (Notion 댓글만)  
- 하루 1회만 알림 등 **중복 방지** (같은 조건으로 여러 번 돌리면 댓글이 여러 번 달릴 수 있음)

---

## 2. Notion 쪽 준비 (담당자 체크리스트)

| 순서 | 할 일 |
|------|--------|
| 1 | [Notion My integrations](https://www.notion.so/my-integrations)에서 **Internal integration** 생성 |
| 2 | **API 키(시크릿)** 복사 — 외부에 노출하지 말 것 |
| 3 | 알림을 줄 **DB 페이지**에서 `⋯` → **Connections** → 해당 통합 **연결** |
| 4 | DB에 아래 [속성 스키마](#3-데이터베이스-속성-스키마)에 맞는 컬럼이 있는지 확인 |

**ID 참고:** 브라우저 주소 `notion.so/.../xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` 에 나오는 **UUID 형태의 ID**가 **데이터베이스 ID**입니다.  
API에서는 `dataSources.query`에 쓰는 **데이터 소스 ID**가 DB ID와 다를 수 있으나, 본 프로젝트는 `databases.retrieve`로 자동으로 맞춥니다. **환경 변수에는 URL의 DB ID**를 넣으면 됩니다.

---

## 3. 데이터베이스 속성 스키마

| 속성 이름 | Notion 타입 | 필수 | 설명 |
|-----------|-------------|------|------|
| `Name` | Title | ✅ | 작업 제목 (코드 기본값, `.env`로 변경 가능) |
| `Due Date` | Date | ✅ | 마감일 (날짜만 있으면 됨) |
| `Status` | Status 또는 Select | 권장 | 값 이름이 `done` 또는 `archive`이면 알림 제외 |
| `Person` | People | 권장 | 비어 있으면 멘션 없이 문구만 |

속성 **이름**을 바꾼 경우에는 `.env`의 `NOTION_*_PROPERTY` 값을 코드 기본값과 맞출 것.

---

## 4. 알림 정책 (언제 댓글이 달리는가)

- **제외:** 마감일 없음 / Status가 `done` 또는 `archive` (대소문자 무시)  
- **포함:** 아래에 해당할 때만 댓글 문구 생성 후 작성  

| 조건 (오늘 기준 마감일과의 일수 차이) | 내용 |
|--------------------------------------|------|
| 3일 남음 | 3일 전 알림 문구 |
| 1일 남음 | 1일 전 알림 문구 |
| 마감 지남 | 연체 일수에 따라 단계별 문구 |

**당일 마감·2일 전·4일 전 등**은 현재 로직에 없어서 댓글이 달리지 않습니다. 정책 변경이 필요하면 `index.js`의 `buildMessage`를 수정합니다.

---

## 5. 실행 환경 비교

| 방식 | 언제 쓰나 | 비고 |
|------|-----------|------|
| **GitHub Actions** | 매일 자동, PC 꺼짐 무관 | **Repository secrets**에 API 키·DB ID 등록 |
| 로컬 `node index.js` | 개발·테스트 | `.env` 사용, Git에 올리지 말 것 |
| 맥 LaunchAgent (선택) | PC 켜진 상태에서만 매일 로컬 실행 | `scripts/` 참고 |

---

## 6. GitHub 설정 (보안)

| 항목 | 올바른 위치 | 잘못된 예 |
|------|-------------|-----------|
| `NOTION_API_KEY`, `NOTION_DATABASE_ID` | **Settings → Secrets and variables → Actions → Repository secrets** | Deploy keys, Environment variables에만 넣고 Actions에서 참조 안 함 |
| 소스 코드 | API 키를 파일에 하드코딩하지 않음 | |

- **Deploy keys**는 Git SSH용이며, Notion API 키와 무관합니다.  
- 저장소가 **Private**이어도 시크릿은 동일하게 설정합니다.

---

## 7. 문제 해결

| 증상 | 확인할 것 |
|------|-----------|
| `object_not_found` | DB에 통합 **Connections** 연결 여부, `NOTION_DATABASE_ID`가 **DB URL의 ID**인지 |
| 댓글이 안 달림 | 해당 날짜에 [알림 정책 §4](#4-알림-정책-언제-댓글이-달리는가)에 맞는 행이 있는지, Status가 `done`/`archive` 아닌지 |
| Actions에서만 실패 | Repository secrets 이름이 정확히 `NOTION_API_KEY`, `NOTION_DATABASE_ID` (대소문자 동일) |
| 로컬만 됨 | Actions 시크릿 누락 또는 키 만료 |
